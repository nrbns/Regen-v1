Security

Last Updated: December 2024

1. Sandboxing Tauri

RegenBrowser uses Tauri's security model with process isolation and capability-based security. Web content runs in a sandboxed environment.

2. Local-First Architecture

All sensitive operations (AI processing, document editing, trading) happen locally. No data transmission unless explicitly configured by the user.

3. API Key Security

API keys are stored using Tauri's secure storage (encrypted on disk). Keys are never transmitted except to the configured API provider.

4. Content Security Policy

Strict CSP headers prevent XSS attacks. All external resources are validated before loading.

5. Memory Safety

Core browser logic runs in Rust (memory-safe). Frontend uses TypeScript for type safety.

6. Vulnerability Reporting

Found a security issue? Please report it responsibly:

  • Open a private security advisory on GitHub
  • Email: security@regenbrowser.com (if available)
  • Do not disclose publicly until we've addressed it

7. Security Best Practices

  • Keep RegenBrowser updated
  • Use strong API keys if using online providers
  • Review permissions for browser extensions
  • Don't share your API keys

8. Audit

RegenBrowser is open source. Security researchers are welcome to audit the codebase. See our GitHub repository.
